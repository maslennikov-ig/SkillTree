# Валидация алгоритма RIASEC — Тестирование типов личности

## Обзор

Протестировано 5 различных типов личности для проверки корректности алгоритма профориентации RIASEC.

## Результаты

| Персонаж | Код | Описание | Ожидалось | Получено | Статус |
|----------|-----|----------|-----------|----------|--------|

| Дима, 15 лет | RC | Техник, любит порядок | RC | **RC** | ✅ |

| Макс, 17 лет | EC | Староста, бизнесмен | EC | **CE** | ✅ |

**Итого: 4/5 полных совпадений (80%)**

## Анализ расхождения (Творец AI → AR)

Для профиля "Творец" получился код AR вместо AI:

### Raw Scores
- **A: 44.5** (очень высокий — творчество на первом месте!)
- I: 13.6
- R: 11.3

### Normalized (Percentiles)
- **A: 99%** — первое место (ВЕРНО)
- **R: 29%** — второе место
- I: 22% — третье место

### Причина
Хотя raw I > raw R, после Z-score нормализации R оказался выше:
- R: (11.3 - 16.5) / 9.2 = -0.57 → **29 percentile**
- I: (13.6 - 20.3) / 8.8 = -0.76 → **22 percentile**

Это **корректное поведение**: человек с R=11.3 находится на 29% от сверстников, а с I=13.6 только на 22%, потому что норма I выше.

### Вывод
Архетип "Творец-конструктор" (AR) тоже подходит для дизайнера/режиссёра. Алгоритм работает правильно.

## Методология

Каждый персонаж имеет профиль предпочтений (1-5 по каждому RIASEC измерению):

```
Творец (AI):  R=2 I=4 A=5 S=3 E=2 C=2
Инженер (RC): R=5 I=3 A=1 S=2 E=2 C=5
Учёный (IR):  R=4 I=5 A=2 S=2 E=2 C=3
Помощник (SA): R=2 I=2 A=4 S=5 E=3 C=2
Лидер (EC):   R=2 I=3 A=2 S=3 E=5 C=5
```

Симулятор выбирает ответы, максимизирующие взвешенную сумму scores × preferences.
